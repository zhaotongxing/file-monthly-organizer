# 文件月整理工具 v1.3.1

自动搜索电脑中的 Office 文档、图片、视频文件，并按照文件的修改时间按月整理到相应的文件夹中。支持关键词搜索过滤、**重复文件检测与删除**、撤销最近一次整理任务、以及按文件类型筛选扫描。

## v1.3.1 更新（重复文件检测与删除）

### 智能去重功能
- **基于内容哈希检测**：使用 MD5 哈希值比对文件内容，精确识别完全重复的文件
- **大文件优化**：超过 4MB 的文件逐块读取计算哈希，避免内存溢出
- **多种保留策略**：
  - **最新修改**（默认）：保留修改时间最新的文件
  - **最早修改**：保留修改时间最旧的文件
  - **最短路径**：保留路径最短的文件（通常意味着更接近根目录）
- **预览模式**：先预览将要删除的文件，确认后再执行实际删除
- **详细报告**：生成包含哈希值、文件路径、释放空间等信息的去重报告

### 使用去重功能

1. 扫描文件后，点击「查找重复」按钮
2. 选择保留策略（最新/最早/最短路径）
3. 等待分析完成（计算文件哈希）
4. 在「重复文件」标签页查看重复组列表
5. 点击「预览删除」查看效果，或「执行删除」实际操作

**注意：** 删除操作不可撤销（除非使用系统回收站恢复），建议先预览再执行。

在「扫描文件类型」区域可以勾选要扫描的文件大类：

- **图片** - JPG、PNG、GIF、RAW、HEIC 等

**注意：** 如果全部取消勾选，程序会默认扫描所有类型。

## v1.2.1 更新（关键词子文件夹）

### 搜索结果智能分层
当勾选「仅整理搜索结果」时，程序会：
1. 在目标目录下创建一个以**关键词命名**的文件夹
2. 在该关键词文件夹内部，再按月创建子文件夹
3. 匹配的文件整理到对应月份的子文件夹中

**示例：** 搜索关键词 `合同`，勾选仅整理搜索结果
```
目标文件夹/
└── 合同/                          ← 关键词子文件夹
    ├── 2024年01月/
    │   └── 合同_A.docx
    ├── 2024年03月/
    │   └── 合同_B.xlsx
    └── 2024年05月/
        └── 合同_C.pdf
```

如果不勾选「仅整理搜索结果」，则直接在目标目录下创建月份文件夹（原有行为）。

**关键词文件夹命名规则：**
- 自动清理非法字符（`< > : " / \ | ? *` 替换为下划线）
- 多个空格/逗号合并为单个下划线
- 过长关键词自动截断至50字符
- 空关键词使用默认名称「搜索结果」

## v1.2.0 功能（任务撤销）

### 撤销整理任务
- **一键撤销**: 点击「撤销上次」按钮恢复最近一次整理操作
- **复制撤销**: 删除目标位置的文件副本
- **移动撤销**: 将已移动的文件移回原位置（同名冲突自动加 `_恢复` 后缀）
- **历史记录**: 自动持久化保存，支持查看和管理所有任务

### 关键词搜索
- **多关键词搜索**: 支持用空格或逗号分隔多个关键词
- **AND / OR 逻辑匹配**: 任一匹配或全部匹配
- **搜索范围**: 文件名+路径 / 仅文件名 / 扩展名 / 完整路径
- **高级选项**: 大小写敏感、正则表达式支持
- **智能整理**: 可选择仅整理搜索结果或全部文件

## 功能特点

- **自动扫描**: 递归扫描指定目录下的所有支持的文档文件
- **智能分类**: 根据文件修改时间自动归类到"YYYY年MM月"文件夹
- **多种操作模式**: 预览、复制、移动
- **文件夹命名**: 支持多种文件夹命名格式
- **完整报告**: 生成详细的操作报告
- **进度显示**: 实时显示扫描和整理进度

## 使用方法

### 界面操作流程

```
1. 选择源文件夹
2. 勾选要扫描的文件类型（Office文档/图片/视频，默认全部）
3. 点击「开始扫描」
4. （可选）输入关键词 → 点击「关键词搜索」
5. 选择目标文件夹
6. 选择操作模式（复制/移动/预览）
7. 点击「开始整理」
8. （如需撤销）点击「撤销上次」→ 确认撤销
```

### 撤销功能使用说明

**何时可以使用撤销？**
- 只有执行了实际文件操作（复制/移动）的任务才能撤销
- 预览模式的任务不可撤销（因为没有实际改变文件）
- 已经被撤销过的任务不能再次撤销

**撤销复制操作会发生什么？**
- 删除目标文件夹中的文件副本
- 原位置的文件完全不受影响
- 空的目标月份文件夹会被自动清理

**撤销移动操作会发生什么？**
- 将文件从目标文件夹移回原位置
- 如果原位置已经有同名文件，会在文件名后自动添加 `_恢复1` 后缀
- 原文件夹如果不存在，会自动创建

**任务历史在哪里查看？**
- 点击「任务历史」标签页
- 可查看所有整理任务的：时间、模式、文件数量、执行状态
- 已撤销的任务会显示 ↩ 标记和撤销时间

### 关键词搜索使用示例

| 场景 | 关键词输入 | 匹配模式 | 搜索范围 |
|------|-----------|---------|---------|
| 找所有合同和报告 | `合同 报告` | OR | 文件名+路径 |
| 找2024年的销售文件 | `2024 销售` | AND | 文件名+路径 |
| 只找PDF文件 | `.pdf` | OR | 扩展名 |
| 找特定年份 | `202[34]` | OR | 文件名+路径（正则） |

## 注意事项

1. **移动操作不可逆（除非撤销）**: 使用"移动文件"模式时，原位置的文件将被移走。但不用担心，整理完成后可以随时点击「撤销上次」恢复。
2. **建议先预览**: 首次使用或不确定时，建议先用"仅预览"模式查看整理计划。
3. **撤销冲突处理**: 如果撤销移动操作时原位置已有同名文件，系统会自动在文件名后添加 `_恢复1` 后缀，避免覆盖。
4. **历史记录文件**: 任务历史保存在程序目录下的 `organizer_history.json` 文件中，请勿手动删除。
5. **权限问题**: 某些系统目录可能需要管理员权限才能访问。

- **threading**: 后台任务处理
